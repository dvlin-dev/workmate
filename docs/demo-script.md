# Workmate 演示剧本（脱敏 · 软件工程师的一周）

> 用途：演示「自然语言录入 → AI 归因 → 看板实时动 → 一键周报」的完整闭环，以及人机协作。
> 内容均为脱敏的通用工程场景，不含任何真实项目隐私。
> 时间锚点：本周（周一 6/22 ~ 周日 6/28），**今天是周三 6/24**。周一/周二的进展是"已发生"，周三是"今天"。
> 演示前确保 app 已加载本周空看板（必要时先用右上角「🗑 清空」或重启 `pnpm dev`）。

主角设定：一个后端/全栈工程师，本周主要在做"订单服务"相关的几件事。照着下面一条条往输入框发即可。

---

## 第 1 条 · 开场（周一计划，建好整个看板）

> 整段发进去 → 建出 3 个目标 + 一串待办；带日期的会写进「提醒事项」。

```
这周的安排同步给你：

时间线：
- 6/25 完成订单服务 v2 重构的 code review
- 6/27 灰度上线 10% 流量
- 6/30 全量上线并接入监控告警

todo：
1、读一遍旧订单服务代码，把依赖关系理清楚
2、把订单和库存的强耦合拆开，先用接口隔离
3、给 v2 核心链路补单元测试，覆盖率到 70%
4、修掉用户反馈的 3 个线上问题
5、接入 Prometheus 指标 + 告警，6/30 前搞定
```

---

## 第 2 条 · 周一晚的进展

```
今天把旧订单服务代码过了一遍，依赖关系基本理清楚了，下单链路里耦合最重的是库存校验那块
```

> 演示点：agent `find_goal` 定位到"读代码/重构"目标，`complete_task` 勾掉对应待办，右侧进度条按完成比例实时上涨（进度是派生值，手动勾选同样会动）。

---

## 第 3 条 · 周二上午

```
订单和库存的依赖拆出来了，库存那边先用接口 mock 顶着，下午开始联调
```

## 第 4 条 · 周二晚

```
线上那个下单偶尔超时的问题定位到了，是数据库连接池没正确释放，已经提了个 fix 等审
```

> 演示点：归因到"修线上问题"目标。可接着发一句「那个超时 bug 修复合并了，先标完成」演示**完成待办**。

---

## 第 5 条 · 周三上午（今天）

```
v2 的单元测试补了一半，核心下单和退款链路覆盖到 60% 了，还差优惠券那条分支
```

## 第 6 条 · 周三下午（今天）

```
监控这块今天起了个头，先把下单成功率和 P99 延迟两个 Prometheus 指标埋上了，告警规则还没配
```

> 演示点：连续两条分别归因到"补测试""接监控"两个不同目标，体现**多目标归因**的准确性。

---

## 第 7 条 · 顺手记一笔

```
对了，还有两个用户反馈的小问题（导出乱码、列表分页错位）今天没顾上，明天集中处理
```

---

## 第 8 条 · 生成周报（演示高潮）

> 真实是周五触发；演示时随时可发，或直接点右下角「一键生成周报」。

```
帮我生成一下这周的周报
```

> 输出"本周完成 / 进展亮点 / 风险与卡点 / 下周计划"四段式 markdown，可一键复制。

---

## 穿插演示「人机协作」（右侧看板手动操作）

- 目标卡片下点 **添加待办**，例如手动加一条「补优惠券分支测试」；
- 「今日聚焦」里**勾掉**已完成的待办（进度/状态实时变）；
- 底部「**+ 新建目标**」手动建一个，例如「整理重构上线 checklist」；
- 这些手动改动同样进周报原料 —— 体现"人 + agent 共同维护这棵周目标树"。

---

## 备选场景（如需换风格）

- 偏前端 / 产品迭代的一周；
- 偏数据 / 算法的一周；
- 更口语化、随手碎碎念式输入（更能体现"低摩擦录入"）。

> 需要时再补；本剧本主要演示后端工程闭环。
